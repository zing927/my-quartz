# 什么是Vibe Coding

Andrej Karpathy 确认“Vibe Coding”被柯林斯词典评为 2025 年度词汇，强调软件开发已进入“意图驱动”时代，开发者正逐渐遗忘代码本身。
https://x.com/karpathy/status/1886192184808149383

尤其是google 发布了最新的gemini 3后，当工具的门槛被彻底打平，当任何一个普通人都能在短时间内、几乎零成本地构建出一个“够用的产品”。

就好像这位博主，通过gemini3的live功能来练习英语。

live功能好比AI有了视野，它可以看到你的真实世界。当AI可以看到什么的时候，你就可以直接可以问它。

# 开发YoutubeToRead插件

自从找到魔法🪜后，对全球好内容的好奇心一直没停过。
但是苦于英语水平太差（考研 35 分，百分制🌚，国家线 34），阅读很吃力。后来有了沉浸式翻译插件，阅读文章就几乎问题了，通过Rss订阅了不少博客。

但是视频还是无能无力，虽然有着字翻译，视频时间太长，有的时候并没有这么大块的时间去看。也尝试过把字幕下载下来，丢给chatgpt让它总结，效率太低。所以就有了想为自己开发一个工具来解决这件事。

下载了cursor和trae都试了一下，折腾了几天，就可以用上了。
目前可以达到的能力，姑且算1.0版本：
打开一个想看的视频，打开插件重写面板，点击开始总结就可以拥有“阅读版”。出结果那一刻，成就感爆棚。
![[images/Pasted image 20251220210356.png]]
![[images/Pasted image 20251220211110.png]]
目前插件不仅支持模型Api、prompt的替换，而且支持根据Markdown的格式，阅读性更优。而且费用超低，请求deepseek一次api只要0.01的费用。

目前插件还有一些不足：
- 总结的浮层没有固定，经常不小心就关闭了。更适合左侧抽屉划出展示。
- 插件名字和icon、整体UI未设计。必须小众而精美。
- 目前生成的结果是文字版，希望后续可以按照一定比例的图片输出

🍉整体感受非常不错，cursor无需🪜就可以使用，更快，但是每日限额。trae免费无限量，够用，更慢一些。没有代码基础，也感受不出两者之间的其他差异。deepseek相当便宜，在4月份买的，接入obsidian到现在还没用完。

唯一的建议是，一开始就描述完整、清晰的需求。从实现的功能，到页面交互，越详细越好。其他就不断尝试吧。
