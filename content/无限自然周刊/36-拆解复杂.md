# 🍉如何拆解复杂系统？

微信官方出品：分享尝试使用 C4 模型对复杂需求进行拆解，帮助设计师理清角色关系与细节，避免遗漏。该模型最大的好处是可以帮助我们将问题与关注点分级，不是一个层面的问题，不要在同个层面解决。
![[images/Pasted image 20240714222758.png]]
C4 模型的结构重点：
1. 定义系统上下文：包括系目标、用户需求、业务领域等。
2. 定义系统功能：描述了系统的整体结构和组织方式。将系统划分为不同的功能模块，例如子系统、二级模块、组件等，并明确它们之间的关系。
3. 定义系统流程：按照功能模块对功能内的流程线做横向关系和顺序，纵向单个节点的层次结构的梳理。
4. 设计具体页面：确定设计目标，转化为具体页面。

# 🍉一个对书籍全面而深入总结的prompt

请帮我对这本书进行全面而详尽的总结,按照以下步骤进行:  
  
1. 全书框架梳理:在开始详细章节总结前,先对全书的整体框架和脉络进行系统梳理,包括:  
a) 主题框架:归纳全书的核心主题和子主题,分析作者是如何通过各个章节来展开和论证这些主题的  
b) 内容走向:概括作者在全书中的思路走向,分析作者是如何层层推进、深化论述的,逻辑链条是如何环环相扣、逐步展开的  
c) 结构特点:分析作者在全书结构安排上的特点和优劣,如开篇、过渡、结尾等环节的设置是否合理有效  
  
2. 章节内容总结:对全书的每一章进行深入和详细的概括,包括:  
a) 核心内容:清晰、全面地总结本章的中心思想、主要内容和结论  
b) 关键论点:提炼作者在本章提出的最重要、最有见地的论点,这些论点可能与主流观点不同,但有新颖的视角  
c) 论点论据:系统梳理作者在本章提出的主要论点,以及用来支撑这些论点的论据、例证等,须引用原文作为论据  
d) 关键概念:列出本章提到的关键概念、术语,并给出简要解释  
e) 延伸思考:提出作者虽未明说,但值得深究和思考的问题,以及可以进一步延伸探讨的话题  
  
3. 案例分析与方法论:选取书中有代表性的案例,用作者提出的方法论逐一分析,看方法论如何指导实践,包括:  
a) 案例梳理:简要概括书中提到的主要案例,厘清每个案例的背景、过程、结果等关键要素  
b) 方法总结:归纳书中介绍的主要方法、工具和技巧,分析其使用条件、操作步骤和预期效果  
c) 案例分析:选取有代表性的案例,运用作者总结的方法论逐一分析,看理论如何指导实践,取得什么效果  
d) 方法评价:评析作者提出的方法的优劣和创新之处,及其适用的条件和范围,并提出可能的改进建议  
  
4. 核心观点提炼:在把握全书整体脉络的同时,提炼作者贯穿全书的核心观点,包括:  
a) 核心论点:明确、准确地概括作者的核心论点,即作者在全书中反复强调、力图证明的最根本观点  
b) 主要论据:归纳作者用来支撑核心论点的主要论据和论证逻辑,引用原文作为支撑材料,评析其充分性和有效性  
c) 创新性与局限性:分析作者的核心观点有何新意和价值,以及存在哪些不足或有待商榷之处  
d) 启示与反思:讨论这些核心观点给你带来哪些启发,引发了你怎样的思考和反思  
  
格式要求:  
1. 使用 Markdown 格式,层次清晰,美观大方。  
具体要求如下:  
- 用 # 表示一级标题(如"全书总结"), ## 表示二级标题(如"核心内容"、"论点论据"等),依此类推  
- 每部分用空行分隔,增加可读性  
- 用 \*、-、+ 等符号表示列表,并用缩进表示层级关系  
- 用 > 表示引用,并在引用文字后用括号注明出处  
用 \\\\\\` 表示代码块或引用大段文本  
2. 字数不限,力求全面详尽,深入透彻。但要注意条理清晰,避免冗长啰嗦,确保可读性。  
3. 引用原文时须注明出处,如章节、页码等。引文和注释用 markdown 格式标记。  
4. 语言要精练准确,逻辑要严密缜密,观点要鲜明深刻,分析要全面客观,尽量避免口语化、随意化的表达。  
5. 在每部分引言时,先用一句话概括该部分的主旨,起到提示和引导作用。  
在每部分结尾处,用一两句话小结概括,加深印象。  
  
其他要求：  
由于你要完成的任务比较复杂，让我们分步骤进行，你每完成一个步骤，让我们达成共识后，再进行下一步。

作者大量阅读以及实践后的个人总结：
![[images/Pasted image 20240714105707.png]]
1. **小步前进**：日常习惯至关重要，微小持续的改变比一次性大变动更能带来长远的成果。日本的“改善”概念和《原子习惯》中1%的逐步提升都强调了这一点。
3. **改变思维导图**：积极思考的力量，通过可视化目标并逆向规划来实现目标。《秘密》等书籍提醒我们，虽然吸引力法则不存在，但规划和过程是关键。
4. **接受挑战**：斯多葛哲学教导我们接受并欢迎最坏的情况，从而减少恐惧。埃莉诺·罗斯福的话鼓励我们每天做一件害怕的事，以培养精神韧性。
5. **避免即时判断**：在做出判断前深呼吸、保持开放心态、设身处地为他人着想，这些都能避免仓促下结论。
6. **记住生命的终结**：意识到生命的有限性可以让我们更专注于重要的事物，避免日常琐事的干扰。
7. **保持趣味性**：即使在面对困难时，也要尝试让事情变得有趣，如Jane McGonigal通过游戏化的方式帮助人们从疾病中恢复。
8. **对他人有用**：黄金法则是自助建议中最古老的一条，强调了对他人有用的重要性。
9. **完美主义与拖延**：完美主义是拖延的根源，接受不完美并开始行动是克服这一问题的关键。
10. **基本生活维护**：睡眠、锻炼、饮食和放松是维护身心健康的基础，需要重复进行。
11. **写下来**：无论是计划、想象还是列清单，写作都是实现目标的重要工具。
12. **阅读的局限性**：阅读是自我提升的一部分，但最终需要将知识付诸实践，并且认识到自己的局限性。
