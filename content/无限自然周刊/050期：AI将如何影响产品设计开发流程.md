# 🍉第二大脑：学习新事物的4个阶段系统

当你遇到一个兴奋的事情，打算也想快速拥有和掌握它。那么到底从哪里开始呢？Tiago Forte 提出学习任何新事物的四个阶段：
- **沉浸 (Immersion)**：初步接触，广泛获取信息，订阅相关新闻、关注KOL，形成初步认知。
- **构建 (Building)**：动手实践，以小项目为目标，在实践中发现知识的不足，快速入门。
- **结构化学习 (Structured Learning)**：找到1-3位导师，专注于他们的经验和指导，避免信息过载。
- **联系 (Connection)**：加入社群，与他人交流，通过分享和协作加速学习。

第一阶段：沉浸式接触

让自己沉浸在新领域中，分配更多的时间，更多的注意力。这种接触式学习可以让我们潜意识自动构建思维模型。
- 播客，视频，newsletter
- 寻求推荐，而非算法推荐，获取高质量信息源

第二阶段：动手实践

知道，不如做一个出来。因为理论和实际存在巨大的偏差。就像马斯克提到的，一个糟糕的开始，就是成功。目标是完成，而不是完美。
- 设定一个截止目标
- 做一个简单真实的项目

第三阶段：找 1-3 个导师

找到导师，系统化学习。初步沉浸式和实践后，你已经比较清楚的识别谁才是真正懂行的人。找 1-3 个导师，值得深挖的导师，不需要从所有人那里学。
- 遵循他们完整的方法，而不是东打一枪，西打一枪
- 用他们的观点筛选信息

第四阶段：联系

建立真实的人际关系和分享。加入一个实践社群，带一个团队，将会指数成长。
- 组织他人一起学习，加速自己成长



# 🍉AI聊天框只是开胃菜，绝不是AI体验的终点
UX专家Amelia Wattenberger尖锐指出的："聊天机器人不是界面的未来"。那只是开发者“最省事的路径”。人们不擅长使用聊天界面，却被迫使用所谓的“提示词”。因为那样脱离了原有的工作流，这是一个低效的交互模式。

作者提出：<u>混合工作空间模型，才是全新的设计框架</u>=原有的工作环境+智能层（The intelligent Layer）将AI能力，以感知上下文的方式，嵌入到工作环境中。它不四一个独立的聊天界面，而是一种能够理解上下文、维持状态、并在不打断心流的情况下提供相关建议的智能辅导。
![[images/Pasted image 20250713162528.png]]
将“智能层”集成到工作流中的方式：
- 内联建议：如同gihbub copilot那样，在用户输入时，直接提供建议。
- 上下文面板：根据用户的操作，在侧边或浮动出现相关的AI功能
- 现有的UI的渐进式增强：对已有的UI元素进行智能化改造
- ![[images/Pasted image 20250622225035.png]]
- 环境智能：AI在后台默默工作，在最需要的时刻才现身。

几个关键原则：
1. **上下文感知 (Contextual Awareness):** AI必须能理解用户正在做什么，以及他们的意图是什么。
2. **渐进式披露 (Progressive Disclosure):** 根据用户的熟练程度和使用场景，逐步揭示高级AI功能，避免信息过载。
3. **工作流集成 (Workflow Integration):** AI能力应该增强用户已有的工作模式，而非强迫他们学习一套全新的流程。
4. **认知负荷降低 (Cognitive Load Reduction):** 设计的目标是让用户专注于他们的核心工作，而不是把精力耗费在如何与AI打交道上。

#  🍉生成式 UI 将往哪个方向发展

这篇文章是妙多的 Co-Founder 昊然对 AI 如何改变 UI 领域的最新总结和感悟。

> 作为 AI 领域的实践者和探索者，三年多来我们投身于 AI 设计工具的打造，见证了生成式 UI 从被专业设计师视作 “玩具”，到如今与 vibe coding 一同改写开发与设计工作流的巨大演变。我们看到一个全新的“需求->代码->设计”工作流正在形成。

如今的生成式AI经过了“套模版”到“理解设计系统”，但还达不到专业设计的效果。这好比外卖送到小区门口能力达到了，但送到家门口的最后 100 米不好解决。
