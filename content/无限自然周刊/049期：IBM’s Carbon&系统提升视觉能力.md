# 🔥系统提升视觉设计的实用指南

Filippos分享了14年的视觉设计经验，可以帮助你系统、有效的提升视觉设计技能。文章非常体系性，可实践性。
- 打好基础
- 研究设计语言
- 积累实战经验
- 培养直觉与设计肌肉记忆
- 识别过去的问题并从中学习
- 挑战不同风格和产品类型

### **打好基础**

在你开始练习之前，理解用户界面中最基本的视觉设计原则非常重要：

- **字体排版**
- **色彩**
- **层级**
- **布局与间距**

根据你自身的背景，你可能需要**深入学习**这些基础，也可能只是需要一个**快速复习和实用技巧**，帮助你理解这些元素如何协同作用于 UI 中。

- **视觉层级** — [https://blog.designary.com/p/sections-structure-and-visual-hierarchy-in-ux](https://blog.designary.com/p/sections-structure-and-visual-hierarchy-in-ux)
- **盒模型** — [https://blog.designary.com/p/layout-basics-the-box-model](https://blog.designary.com/p/layout-basics-the-box-model)
- **间距系统** — [https://blog.designary.com/p/spacing-systems-and-scales-ui-design](https://blog.designary.com/p/spacing-systems-and-scales-ui-design)
- **色彩** — [https://www.interaction-design.org/literature/article/ui-color-palette](https://www.interaction-design.org/literature/article/ui-color-palette)
- **字体排版** — [https://www.interaction-design.org/literature/topics/typography](https://www.interaction-design.org/literature/topics/typography)

### **研究设计语言**

大量的知识可以通过研究主流设计语言获得。这将帮助你深入理解操作系统如何定义设计规范，以及它们如何使用交互模式和组件。

**设计语言推荐：**

- Apple Human Interface Guidelines — [https://developer.apple.com/design/human-interface-guidelines](https://developer.apple.com/design/human-interface-guidelines)
- Material 3 — [https://m3.material.io/](https://m3.material.io/)

**你需要关注的重点包括：**

- **理解它们如何定义字体和色彩系统**
- **理解布局与间距的规范**
- **理解导航结构的设计逻辑**
- **熟悉常用组件：**如标签栏、按钮、表单、表格、提示框、横幅、底部弹窗等
- **掌握常见交互模式：**比如，什么时候使用模态框 vs. 全屏页面？


### **积累实战经验**

想要快速提升视觉设计能力，**最有效的方法之一就是阅读一些实用为主、而非理论为主的设计书籍**。我刚开始做产品设计时，几乎完全依赖试错法。虽然有效，但我花了很多年才意识到自己做错了什么，又该如何改进。现在，好在有很多实用资源能让你快速掌握关键方法和技巧，少走弯路。

**我最推荐的书籍是《Practical UI》by Adham Dannaway。**它非常易读，内容短小精悍，提供了大量 UI 实践技巧，包括常见错误和改进方法。

其他推荐的 UI 设计书籍（可按需选择）：

- Practical UI — [https://www.practical-ui.com?aff=eEnA8](https://www.practical-ui.com?aff=eEnA8)
- Refactoring UI — [https://www.refactoringui.com/](https://www.refactoringui.com/)
- The Design Manual — [https://uiadrian.gumroad.com/l/design-manual](https://uiadrian.gumroad.com/l/design-manual)


### **培养直觉与设计肌肉记忆**

尤其是在职业早期阶段，**即使你掌握了设计基础，也未必积累了足够的“设计直觉”或“肌肉记忆”。**

你可能会：

- **在间距或对齐上犯错**
- **混合使用不一致的文本样式**
- **选择不协调的配色**

**建立“设计肌肉记忆”的最佳方式就是有意识地重复练习。**

### 💡 方法：

1. 打开一个设计灵感网站（强烈推荐：**Mobbin、Appshots、Pageflows**）
2. 找一个你喜欢的产品，下载其界面截图，最好是一个完整流程
3. 把它们贴到 Figma 中，**从头复刻一遍**
4. 留意：布局、间距、色彩、字体、微妙细节等
5. **每周练习一次**，不同类型（移动端 vs. 网页端，消费类 vs. 企业类）

**灵感网站推荐：**

- Mobbin — [https://mobbin.com/](https://mobbin.com/)
- Appshots — [https://appshots.design/](https://appshots.design/)
- Pageflows — [https://pageflows.com/](https://pageflows.com/)
- Nicelydone — [https://nicelydone.club/](https://nicelydone.club/)

💡 **实用技巧：** 每周都用一个灵感网站来浏览和理解新趋势，再结合一本视觉设计类图书深入学习技巧。


### **改造你的旧作品**

提升视觉设计能力的一大关键点是：**识别过去作品的不足并从中学习。**
没有什么比重新审视你 3～6 个月前做的设计，并对其改进，更能锻炼你的设计判断力了。

### 💡 方法：

- **每隔 3 个月**，回顾一次自己的旧作品
- 思考：**哪里不对？现在会怎么做？**
- 结合你在前面练习中积累的知识进行判断：
    - 间距是否一致？
    - 是否用了太多字体或配色风格？
    - 是否有品牌感和愉悦感？
    - 是否有更好的层级或布局方式？


### **挑战不同风格和产品类型**

最优秀的视觉设计师都会不断探索新的风格。然而，在一个单一产品或设计系统的限制下，要拓展风格非常困难。如果你在公司做全职设计师，你能自由探索的空间是有限的。

**解决方案：**

- 做自由职业者，参与不同产品和行业的项目
- 如果不能接私活，也可以：
    - **为一个你喜欢的产品做概念重设计**
    - **从零构思一个概念产品**
    - **参与设计挑战练习**


### **可以探索的方向：**

- 企业 vs. 消费类产品
- 亲切 vs. 严肃的设计风格
- 插画 vs. 照片主导
- 移动端 vs. 网页端产品
- 极简 vs. 内容密集型界面
- 不同垂直行业和场景

💡 **实用技巧：** 每周留出一些时间做“和你日常完全不同”的设计探索。可以是虚构产品、现实产品重设计，或参与公开挑战。

# 🚫结构化拖延

Marc Andreessen提出一个解决拖延症的巧妙方法，核心思想是不要对抗拖延，而是利用拖延的时间完成其他有价值的任务。

> 有的时候遇到了重要的事情，尤其是比较“难办”的事情，我也会倾向于逃避转而选择处理那些没有重要也不紧急的事情。似乎完成了这些事，心理就宽慰一点。

因此巧妙的安排todo清单上的任务，可以让逃避去做的【次要】的事仍有意义。

#  🍉IBM’s Carbon

惊喜、兴奋！Carbon拥有比较完善的AI使用文档。它包含一个专门面向AI的部分——“Carbon for AI”，其中涵盖了组件、设计模式和指南，帮助用户识别AI生成的内容，协助理解产品中AI的使用方式。

Carbon for AI是在现有组件基础上构建的——为了突出AI的存在，它为相关组件添加了蓝色光晕和渐变效果。目前已经有12个组件提供了AI版本，例如弹框、数据表格和文本输入框等。
![[Pasted image 20250622225035.png]]
尽管这些AI版本的组件在视觉上有明确的区别处理，但在实际使用场景中，很难区分哪个组件是当前激活的（因为它们看起来都像是激活状态）。

在下面这个表单中，大多数输入字段是由AI自动填写的，因此使用了AI版本的组件。这些组件即便处于默认状态，也会显示蓝色渐变和边框——这使得用户难以判断当前真正处于激活状态的是哪个字段。
